# MoriiCard — 术语表

- **双语分工 (Bilingual split)** — skill 文件的语言纪律:规则散文一律英文(利指令遵从);中文只允许四种功能性角色——触发词汇、卡面输出文案、CJK 规格单位(如 ≤7字)、中文输出示例。「统一成单一语言」是伪命题,功能性中文不可译。

- **双档 (Dual mode)** — 生成质量两档：快速档（交互即时，精简）/ 精致档（后台或要求精致，完整）。
- **壳预算 (Shell budget)** — 仅计 CSS/SVG/JS 结构的体积上限；数据内容不计入、不可为省壳砍数据。
- **中性表面 (Neutral surface)** — 卡面与页面底永远是固定中性色（纯白卡/近黑卡），任何情绪色相不得染表面。
- **图表色对 (Accent pair)** — 每个内容领域一对高饱和 accent，只进图表几何、chip、微图标；情绪色板的继任者。
- **鲜艳小面积 (Vivid-small)** — 配色原则：高饱和（S 55–75）但单元素面积小、总彩色面积 ≤ 卡面 10%，靠纯中性底压住。
- **60-30-10** — ≥60% 中性表面/留白、~30% 灰调次级、≤10% accent。
- **解剖公式 (Anatomy)** — 数据卡默认结构：图标章+灰标题 → 英雄数字+delta chip → 一句灰上下文 → 图表区(40–60%) → 微轴标。
- **文字原子 (Text atoms)** — 数据卡文字仅以原子存在：标题≤3词 / 数值 / 标签 / chip≤7字 / 每区至多一句微文案；段落仅编辑部变体可用（双标准）。
- **数值行 (Stat row)** — 2–4 个指标并排：墨色大数值+下方灰标签，无盒、共基线。
- **数字块 (Number block)** — 灰标签 + 中性大数值 + 上下文行（chip/旧值参照）；禁止 "0→742" 融合写法。
- **观点条 (Point row)** — 图标/圆点 + ≤7字短语 + 右对齐证据数值；用图形原子取代观点段落。
- **内嵌块 (Inset block)** — 凹陷感中性灰块，承载引述/建议/次级组；取代彩色框与边框。
- **高亮单柱 (Highlight bar)** — 柱群灰化、唯一焦点柱用 accent + 数值标牌；图表叙事核心手法。
- **反蠢规则 (Anti-dumb rules)** — 针对交互事故的硬规则与标准片段：死按钮禁令、弹层 class 切换三出口、tab 状态单源。
- **数据内嵌优先 (Embed-first)** — 已收集数据直接写入 HTML；仅明确要求实时刷新才 fetch，快照兜底。
- **开放布局 (Open layout)** — 数据直接坐卡面、留白分隔；孤数不入盒。
- **角标 (Claim ref)** — 观点条旁的微型编号索引：accent 微染示意可点，悬停浮出"查看依据"提示，点击开依据弹层；与弹层条目同号对应。
- **来源章 (Source marks)** — 引用来源的最终形态：弹层内单个 SVG 字母圆章行，点击直达网站；来源永不上卡面。
- **密度阀 (Density valve)** — 单卡面 >2 主题或 >8 数据行时必须拆成 2–3 个分段药丸 tab；禁止硬塞与无限滚动。
- **图表完整性 (Chart integrity)** — 图表独占整行宽、SVG 固定像素高；窄屏宁可堆叠或砍项，绝不缩放变形、绝不横向滚动。
- **仪表盘布局 (DASH)** — 多领域数据的组件化布局：每个组件是跑完整解剖公式的子卡，各持独立图表色对；表面仍中性。「单卡一色对」在此放宽为「单组件一色对」。
- **图表片段库 (Chart library)** — 图表几何的预验证片段集（CHARTS.md）：锁坐标数学与对齐，不锁构图创意。
- **几何锁定构图自由 (Geometry locked, composition free)** — 片段库的边界原则：会出事故的部分（坐标、居中、比例）必须照抄；数据映射、组合、风格演化必须自由。片段是地板不是天花板。
- **动效常量 (Motion constant)** — 全卡统一的非线性缓动曲线（--ez）；一切动画为一次性入场，限几何安全通道（透明度/变换/描画），尊重减弱动效偏好。
- **锚注 (Anchor annotation)** — 图表内单点强调：峰值点 + 紧贴数值标签，取代图例式说明。
- **结论行 (Insight line)** — 每卡至多一句的「裁决」原子：图表给证据，结论行给答案；说结论，永不复述图表。语义高光，视觉高光的上位替代。
- **告警态 (Alert state)** — 仅当数值越过真实阈值时的琥珀/红强调（chip + 锚注）；无阈值不告警，永不装饰。
- **微卡 (MICRO)** — 交互回应级卡：图标 + 单结论原子 +（真有抉择时）动作行；抉择行走回传通道（选项瓦片），剪贴板回声降级兜底。
- **回传通道 (LIVE channel)** — 跨模式能力,非模式：写卡前用后台一次性本地监听器（python stdlib,127.0.0.1 随机高位端口）,卡内按钮 no-cors beacon 一击即唤醒 agent 续答;仅交互会话、仅真实抉择点（模式问询/方案选择/升级/深入）开启,永不装饰;前台预检（python3 存在 + 端口可绑定）不过 → 整卡不带 LIVE 标记;fetch 失败落回剪贴板回声。
- **选项瓦片 (Option tile)** — 抉择选项的标准形态：36px 微染图标章 + 600 字重标签 + 灰 meta,≥44px 可点;光秃按钮行的上位替代。
- **自替换 (Self-replace)** — 问询/报价卡被点击后进入 #w 哈希等待态自刷,agent 把成品卡原子覆盖（tmp+mv）同一文件;一张卡走完「问→答→成品」,不留废卡。
- **步进布局 (Stepper)** — 「≥3 步有序依赖、每步携富内容（命令/代码/检查点）」数据形状的布局；DASH 同源的形状自动匹配,非模式。进度头复用时间线原子（已过/当前实心 accent、当前加环、未来灰,点可直跳）,步骤体逐 panel 切换（tab 同源单源状态）,底部上一步/下一步（≥44px）+ 计数。步骤体严守文字原子:步骤标题 + 至多一句操作指令 + 内嵌块 + 观点条,一步装不下即拆步,>8 步并子步。内容全内嵌纯卡内导航,回传通道仅按既有规则挂真实抉择点。步骤仅标签级内容或 <3 步 → 时间线/观点条够用,不升级。

- **实验卡 (LAB)** — 可操作实验卡：用户操纵变量、实时读数对比的单文件实验台；回答「取决于场景」类问题的动手形态。
- **提案制 (Propose-first)** — 实验卡的触发纪律：三条件全满足（答案取决于参数 / 单文件可忠实重现 / 有可操纵变量）才在正常回答末尾附一句量化提案，经同意才建；显式索要 demo 直接建；同主题只提一次，被拒即止。
- **实验台 (Stage)** — 实验卡主区：全宽可操作区域，几何可变（实验本体），内嵌操作提示；运行时循环动画仅此处放行。
- **模式问询 (Mode probe)** — 档位信号模糊（交互式但像报告级）时，收集数据之前先问一次：快速版或精致版——以 LIVE 选择卡发出,点击即回传,成品卡自替换同一文件;信号明确时绝不问。
- **升级报价 (Upgrade offer)** — FAST 卡确有内容被扣留时，渲染后唯一允许的一行话：量化说明未展开的 N 项/M 图并提供「精致」口令；通道开着时改为卡上一枚静默按钮（发送 rich,自替换）；无扣留则沉默。
